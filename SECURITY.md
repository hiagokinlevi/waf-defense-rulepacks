# Security Policy

## Supported Versions

This project is currently in bootstrap phase. All content in the `main` branch is considered the supported version.

| Version | Supported |
|---|---|
| main branch | Yes |
| older tags | No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project — including errors in WAF rules that could lead to bypass, or issues in the validator scripts — please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to Report

1. Email: `security@[maintainer-domain]` *(update with actual contact before production use)*
2. Or open a [GitHub Security Advisory](https://github.com/hiagokinlevi/waf-defense-rulepacks/security/advisories/new) via the repository's Security tab.

### What to Include

- A clear description of the vulnerability
- Steps to reproduce or a proof-of-concept
- The affected file(s) and pack(s)
- Potential impact

### Response Timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 5 business days
- **Resolution or mitigation plan**: within 30 days for critical/high severity

### Out of Scope

- Vulnerabilities in applications you are trying to protect using these packs (that is out of scope for this repository)
- General questions about WAF deployment

Thank you for helping keep this project and the community safe.
